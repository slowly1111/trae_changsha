# 计划：构建《熔炉 2025》情感转化应用

## 1. 项目初始化与环境搭建
- **创建项目**: 使用 Next.js 14 (App Router) + TypeScript + Tailwind CSS。
- **安装依赖**:
    - `framer-motion`: 用于核心动画（粒子、震动、浮现）。
    - `lucide-react`: 图标库。
    - `clsx`, `tailwind-merge`: 样式工具。
- **资源准备**: 在 `public/audio` 目录下创建音频文件的占位符（用户后续可替换为真实的 Suno 音乐）。

## 2. 核心架构与状态管理
设计一个主控组件 `FurnaceContainer` 来管理四个阶段的状态流转：
1.  **Idle**: 等待输入。
2.  **Build-up**: 长按蓄力（红光、震动、音效渐强）。
3.  **Collapse**: 熔断瞬间（粒子化、黑屏）。
4.  **Rebirth**: 结果展示（治愈文案、音乐播放）。

## 3. 关键功能实现
### 3.1 前期交互 (The Build-up)
- 实现 `LongPressButton` 组件：
    - 监听按住事件，驱动进度条或热力值。
    - 使用 CSS/Framer Motion 实现屏幕震动 (`Visual Jitter`)。
    - 文本框文字随长按时间变色（白 -> 橙 -> 亮白）。
    - 播放并渐变“燃烧”音效音量。

### 3.2 中期特效 (The Collapse)
- 实现文字粒子化效果：
    - 利用 Framer Motion 的 `layoutId` 或 Canvas 模拟文字碎裂。
    - 粒子向上飘散并淡出。
    - 背景瞬间切换为全黑。

### 3.3 后端逻辑 (The Logic)
- 创建 `app/api/analyze/route.ts`：
    - 模拟/接入豆包 API 逻辑。
    - 接收文本，返回情感分类（焦虑/遗憾/愤怒）和治愈文案。
    - 即使没有真实 API Key，也会提供 Mock 数据以保证流程通畅。

### 3.4 后期重生 (The Rebirth)
- 根据情感类型自动播放对应的背景音乐。
- 电影字幕式逐行浮现治愈文案。
- 展示“2026 灵魂关键词”卡片。
